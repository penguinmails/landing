# Internationalization (i18n)

This landing page supports automatic language detection for **English** (`en`) and **Spanish** (`es`).

## How It Works

Language is determined **automatically** — there is no manual language switcher in the UI.

### Detection Flow

1. A user visits the site (e.g. `https://penguinmails.com/`).
2. The [middleware](../src/middleware.ts) reads the browser's `Accept-Language` header via Astro's `preferredLocale`.
3. If the preferred locale is `es`, the user is redirected to `/es/`.
4. Otherwise, the user stays on `/` (English, the default).

This only triggers on the **root path** (`/`). Direct visits to `/es/pricing` or `/features` are served as-is.

### Adding a New Language

1. Add the locale code to `astro.config.mjs` → `i18n.locales`.
2. Add the language label to `src/i18n/ui.ts` → `languages`.
3. Create a new folder under `src/i18n/locales/{lang}/` with the section files (`common.ts`, `home.ts`, etc.).
4. Create the barrel `index.ts` in that folder.
5. Import and add it to `src/i18n/ui.ts` → `ui`.
6. Create the page routes under `src/pages/{lang}/`.

### Adding a New Translation Key

1. Add the key to [src/i18n/types.ts](../src/i18n/types.ts) in the appropriate interface.
2. Add the English value in `src/i18n/locales/en/{section}.ts`.
3. Add the Spanish value in `src/i18n/locales/es/{section}.ts`.
4. TypeScript will error if any language is missing the key.

## Changing Language Programmatically

Since there is no UI toggle, language can be changed programmatically via the **Web API** or by navigating directly to the locale-prefixed URL.

### Option 1: Navigate to the locale URL

```javascript
// Switch to Spanish
window.location.href = '/es/';

// Switch to English (default, no prefix)
window.location.href = '/';

// Switch the current page to Spanish
const path = window.location.pathname;
window.location.href = `/es${path}`;
```

### Option 2: Override the browser language (for testing)

In Chrome DevTools:
1. Open **Settings** → **Preferences** → **Languages**.
2. Add `es` (Spanish) and move it to the top.
3. Reload the page — the middleware will detect it and redirect.

In Firefox:
1. Navigate to `about:config`.
2. Search for `intl.accept_languages`.
3. Set the value to `es, en` (Spanish first).
4. Reload.

### Option 3: Use the `Accept-Language` header directly

```bash
# Test Spanish detection
curl -H "Accept-Language: es" -I https://penguinmails.com/

# Should return a 302 redirect to /es/
```
