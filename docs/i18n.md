# Internationalization (i18n)

## What This Means

The landing site serves English and Spanish audiences. Language selection is automatic — there is no manual switcher in the UI. Visitors are routed to the right locale based on their browser preference, keeping the experience seamless without requiring any user action.

## How It Works

When a user hits the root path (`/`), the middleware reads the browser's `Accept-Language` header via Astro's `preferredLocale`. If the preferred locale is `es`, the user is redirected to `/es/`. All other visitors stay on `/` (English, the default). Direct visits to locale-prefixed paths like `/es/pricing` are served as-is without re-evaluation (`src/middleware.ts`).

Translation keys are typed. Adding a key to `src/i18n/types.ts` without providing values in all locale files will produce a TypeScript error, so missing translations surface at build time rather than at runtime.

## Implementation Notes

The i18n configuration lives in `astro.config.mjs` under `i18n.locales`. Translation content is organized by locale and section under `src/i18n/locales/{lang}/`, with a barrel `index.ts` per locale. The `useTranslations` and `getLangFromUrl` helpers in `src/i18n/utils.ts` are the primary interface for consuming translations in components and pages.

### Adding a new language

1. Add the locale code to `astro.config.mjs` → `i18n.locales`.
2. Add the language label to `src/i18n/ui.ts` → `languages`.
3. Create `src/i18n/locales/{lang}/` with section files (`common.ts`, `home.ts`, etc.) and a barrel `index.ts`.
4. Import and register the new locale in `src/i18n/ui.ts` → `ui`.
5. Add page routes under `src/pages/[...locale]/`.

### Adding a new translation key

1. Add the key to the appropriate interface in `src/i18n/types.ts`.
2. Add the English value in `src/i18n/locales/en/{section}.ts`.
3. Add the Spanish value in `src/i18n/locales/es/{section}.ts`.

TypeScript will surface any locale missing the key before the build completes.

### Testing locale detection

Override the browser language in DevTools (Chrome: Settings → Preferences → Languages; Firefox: `about:config` → `intl.accept_languages`) or test the redirect directly:

```bash
curl -H "Accept-Language: es" -I https://penguinmails.com/
# Expect: 302 → /es/
```

To switch locale programmatically, navigate to the locale-prefixed URL:

```javascript
window.location.href = '/es/';   // Spanish
window.location.href = '/';      // English (default, no prefix)
```
